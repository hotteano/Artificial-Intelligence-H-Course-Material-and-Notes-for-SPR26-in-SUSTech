\section{Introduction}

In this report, we present a heuristic approach and a genetic algorithm to solve the Information Exposure Maximization Problem (IEMP) in social networks. The IEMP aims to identify a set of seed nodes in a social network that maximizes the expected number of nodes that are exposed to both campaigns or that are not exposed to any campaigns under a given budget constraint. We first describe our heuristic approach, which is based on the concept of node centrality and influence spread. Then, we introduce a genetic algorithm that iteratively evolves a population of candidate solutions to find an optimal or near-optimal set of seed nodes.

\section{IEMP} 

\subsection{Problem Formulation}

The IEMP can be formally defined as follows: 

\begin{definition}
Information Exposure Maximization Problem (IEMP, 2 campaigns version): Given a social network represented as a graph \( G = (V, E) \), where \( V \) is the set of nodes and \( E \) is the set of edges, and two campaigns \( A \) and \( B \), the goal is to find a set of seed nodes \( S \subseteq V \) with a budget constraint \( |S| \leq k \) that maximizes 
$$
\mathbb{E}[V\setminus (r(S_1 \cup I_1) \Delta r(S_2 \cup I_2))], |S_1| + |S_2| \leq k, 
$$
where $S_1, S_2 \subseteq V$ is the target seed set for campaign $A$ and $B$ respectively, $I_1, I_2$ is the initial seed set for campaign $A$ and $B$ respectively, and $r(S)$ is the set of nodes that are \textit{reached} by the campaign with seed set $S$. Note that a node is considered reached if it is (1) a seed node, (2) successfully activated during propagation, or (3) attempted to be activated but failed. The operator $\Delta$ denotes the symmetric difference between two sets, which means that we want to maximize the expected number of nodes that are either exposed to both campaigns or not exposed to any campaign.
\end{definition}

The problem is a NP-hard combinatorial optimization problem, and finding an exact solution is computationally infeasible for large networks. Therefore, we propose heuristic and genetic algorithms approaches to find approximate solutions.

\subsection{Heuristic Approach}

We propose a greedy algorithm based on the LFA (Last-to-First Allocating) strategy to minimize the symmetric difference between the two campaigns' influence spreads.

The pseudocode of the algorithm is as Algorithm \ref{alg:greedy}.

\begin{algorithm}
\caption{Greedy Algorithm for Symmetric Difference Minimization}
\label{alg:greedy}
\begin{algorithmic}[1]
\State $V_{avail} \gets V \setminus (I_1 \cup I_2)$
\State $S_1 \gets \emptyset$, $S_2 \gets \emptyset$
\For{$i = 1$ to $k$}
    \State $\textit{best\_gain} \gets -\infty$, $\textit{best\_node} \gets \text{null}$, $\textit{best\_for\_S1} \gets \text{true}$
    \For{each $v \in V_{avail}$}
        \State $\textit{gain}_1 \gets \textsc{ComputeGain}(v, S_1, S_2, I_1, I_2, \text{for\_S1}=\text{true})$
        \If{$\textit{gain}_1 > \textit{best\_gain}$}
            \State $\textit{best\_gain} \gets \textit{gain}_1$, $\textit{best\_node} \gets v$, $\textit{best\_for\_S1} \gets \text{true}$
        \EndIf
        \State $\textit{gain}_2 \gets \textsc{ComputeGain}(v, S_1, S_2, I_1, I_2, \text{for\_S1}=\text{false})$
        \If{$\textit{gain}_2 > \textit{best\_gain}$}
            \State $\textit{best\_gain} \gets \textit{gain}_2$, $\textit{best\_node} \gets v$, $\textit{best\_for\_S1} \gets \text{false}$
        \EndIf
    \EndFor
    \If{$\textit{best\_for\_S1}$}
        \State $S_1 \gets S_1 \cup \{\textit{best\_node}\}$
    \Else
        \State $S_2 \gets S_2 \cup \{\textit{best\_node}\}$
    \EndIf
    \State $V_{avail} \gets V_{avail} \setminus \{\textit{best\_node}\}$
\EndFor
\State \textbf{return} $S_1, S_2$
\end{algorithmic}
\end{algorithm}

The gain computation using LFA estimation is as Algorithm \ref{alg:computegain}.

\begin{algorithm}
\caption{ComputeGain using LFA Estimation}
\label{alg:computegain}
\begin{algorithmic}[1]
\Require Node $v$, current sets $S_1, S_2$, initial sets $I_1, I_2$, boolean $\text{for\_S1}$
\Ensure Marginal gain of adding $v$ to $S_1$ or $S_2$
\State $\textit{curr} \gets \textsc{EstimateSymDiff}(S_1, S_2, I_1, I_2)$
\If{$\text{for\_S1}$}
    \State $\textit{new} \gets \textsc{EstimateSymDiff}(S_1 \cup \{v\}, S_2, I_1, I_2)$
\Else
    \State $\textit{new} \gets \textsc{EstimateSymDiff}(S_1, S_2 \cup \{v\}, I_1, I_2)$
\EndIf
\State \textbf{return} $\textit{curr} - \textit{new}$
\end{algorithmic}
\end{algorithm}

\begin{algorithm}
\caption{EstimateSymDiff using LFA}
\label{alg:estimatesymdiff}
\begin{algorithmic}[1]
\Require Sets $S_1, S_2$, initial sets $I_1, I_2$
\Ensure Expected symmetric difference estimation
\State $\textit{prob}_1 \gets \textsc{LFAEstimate}(S_1 \cup I_1, \text{campaign}=1)$
\State $\textit{prob}_2 \gets \textsc{LFAEstimate}(S_2 \cup I_2, \text{campaign}=2)$
\State $\textit{sym\_diff} \gets 0$
\For{each $v \in V$}
    \State $p_1 \gets \textit{prob}_1[v]$, $p_2 \gets \textit{prob}_2[v]$
    \State $\textit{sym\_diff} \gets \textit{sym\_diff} + p_1(1-p_2) + (1-p_1)p_2$
\EndFor
\State \textbf{return} $\textit{sym\_diff}$
\end{algorithmic}
\end{algorithm}

\begin{algorithm}
\caption{LFAEstimateReach}
\label{alg:lfaestimate}
\begin{algorithmic}[1]
\Require Seed set $S$, campaign index $c$
\Ensure Reached probability $\textit{reach}[v]$ for each node $v \in V$
\For{each $v \in V$}
    \State $\textit{reach}[v] \gets 0$, $\textit{active}[v] \gets 0$
\EndFor
\For{each $s \in S$}
    \State $\textit{reach}[s] \gets 1$, $\textit{active}[s] \gets 1$
\EndFor
\For{$\textit{iter} = 1$ to $\textit{max\_iter}$}
    \State $\textit{new\_reach} \gets \textit{reach}$, $\textit{new\_active} \gets \textit{active}$
    \For{each $u \in V$ where $\textit{active}[u] > 0$}
        \For{each $(u, v, p_1, p_2) \in E$}
            \State $p \gets p_1$ if $c=1$ else $p_2$
            \If{$\textit{active}[u] > \epsilon$}
                \State \Comment{Node $v$ is reached if any active parent attempts to activate it}
                \State $\textit{new\_reach}[v] \gets 1 - (1 - \textit{active}[u]) \times (1 - \textit{new\_reach}[v])$
            \EndIf
            \State \Comment{Node $v$ is activated if any parent succeeds}
            \State $\textit{contribution} \gets \textit{active}[u] \times p$
            \If{$\textit{contribution} > \epsilon$}
                \State $\textit{new\_active}[v] \gets 1 - (1 - \textit{contribution}) \times (1 - \textit{new\_active}[v])$
            \EndIf
        \EndFor
    \EndFor
    \State $\textit{reach} \gets \textit{new\_reach}$, $\textit{active} \gets \textit{new\_active}$
\EndFor
\State \textbf{return} $\textit{reach}$
\end{algorithmic}
\end{algorithm}

The algorithm is implemented in Python in the file \texttt{IEMP\_heur.py}. 

\paragraph{Key Distinction: Reached vs. Active Nodes} According to the IEMP problem definition, a node is considered \textit{reached} (or \textit{exposed}) by a campaign if it is either (1) a seed node, (2) successfully activated, or (3) \textit{attempted to be activated but failed}. This is a crucial distinction from standard influence maximization problems where only successfully activated nodes count. Therefore, our estimation must track two quantities: the probability that a node is \textit{reached} (exposed to the campaign), and the probability that a node is \textit{activated} (can propagate further).

\paragraph{Rationale} The optimization target is to minimize the expected symmetric difference between the two campaigns' \textit{reached} node sets (not just activated nodes). The algorithm greedily selects nodes that maximize the reduction of symmetric difference. Specifically, for each candidate node, we estimate its marginal contribution using the LFA (Last-to-First Allocating) strategy with the following key features:
\begin{itemize}
    \item \textbf{Dual tracking:} We simultaneously track $\textit{reach}[v]$ (probability $v$ is exposed to the campaign) and $\textit{active}[v]$ (probability $v$ is activated and can propagate).
    \item \textbf{Reached propagation:} A node $v$ becomes reached if any active parent attempts to activate it, regardless of success (using OR-formula: $1 - \prod(1 - p_{\text{parent}})$).
    \item \textbf{Active propagation:} A node $v$ becomes active only if the activation attempt succeeds (using OR-formula over all parents' success probabilities).
\end{itemize}
This approach efficiently approximates the expected influence spread under the IEM definition without expensive Monte Carlo simulations. The node with the highest marginal gain in reducing symmetric difference is selected and added to either $S_1$ or $S_2$, whichever yields better improvement.
