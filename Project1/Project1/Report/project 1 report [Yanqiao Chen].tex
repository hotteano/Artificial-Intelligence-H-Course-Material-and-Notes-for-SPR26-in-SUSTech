\section{Introduction}

In this report, we present a heuristic approach and a genetic algorithm to solve the Information Exposure Maximization Problem (IEMP) in social networks. The IEMP aims to identify a set of seed nodes in a social network that maximizes the expected number of nodes that are exposed to both campaigns or that are not exposed to any campaigns under a given budget constraint. We first describe our heuristic approach, which is based on the concept of node centrality and influence spread. Then, we introduce a genetic algorithm that iteratively evolves a population of candidate solutions to find an optimal or near-optimal set of seed nodes.

\subsection{Problem Formulation}

The IEMP can be formally defined as follows: 

\begin{definition}
Information Exposure Maximization Problem (IEMP, 2 campaigns version): Given a social network represented as a graph \( G = (V, E) \), where \( V \) is the set of nodes and \( E \) is the set of edges, and two campaigns \( A \) and \( B \), the goal is to find a set of seed nodes \( S \subseteq V \) with a budget constraint \( |S| \leq k \) that maximizes 
$$
\mathbb{E}[V\setminus (r(S_1 \cup I_1) \Delta r(S_2 \cup I_2))], |S_1| + |S_2| \leq k, 
$$
where $S_1, S_2 \subseteq V$ is the target seed set for campaign $A$ and $B$ respectively, $I_1, I_2$ is the initial seed set for campaign $A$ and $B$ respectively, and $r(S)$ is the set of nodes that are exposed to the campaign with seed set $S$. The operator $\Delta$ denotes the symmetric difference between two sets, which means that we want to maximize the expected number of nodes that are either exposed to both campaigns or not exposed to any campaign.
\end{definition}

The problem is a NP-hard combinatorial optimization problem, and finding an exact solution is computationally infeasible for large networks. Therefore, we propose heuristic and genetic algorithms approaches to find approximate solutions.

\subsection{Heuristic Approach}

We apply the IMRanking algorithm to optimize this problem. 

The psuedocode of the algorithm is as Algorithm \ref{alg:IMRanking}.

\begin{algorithm}
\caption{IMRanking Algorithm for IEMP}
\label{alg:IMRanking}
\begin{algorithmic}[1]
\State $r^{(0)} = r$
\State $t \gets 0$
\Repeat
        \State $t \gets t + 1$
        \State Calculate $M_{r^{(0)}}$ with respect to ranking $r$
        \State Generate a new ranking $r^{(t)}$ by sorting nodes in decreasing order according to $M_{r^{(t)}}$
\Until{$r^{(t)} = r^{(t-1)}$}
\State \textbf{return} $r^{(t)}$
\end{algorithmic}
\end{algorithm}

The Calculation of $M_{r^{(0)}}$ is as Algorithm \ref{alg:calM}.

\begin{algorithm}
\caption{Calculation of $M_{r^{(0)}}$}
\label{alg:calM}
\begin{algorithmic}[1]
\For{$i=1$ to $n$} 
    \State $M_r(v_{r_i}) \gets 1$
\EndFor
\For{$i = n$ to $2$} 
    \For{$j= 1$ to $i-1$} 
        \State $M_r(v_{r_j}) \gets M_r(v_{r_n}) + p(v_{r_j}, v_{r_i}) \times M_r(v_{r_i})$
        \State $M_r(v_{r_i}) \gets (1-p(v_{r_j}, v_{r_i})) \times M_r(v_{r_i})$
    \EndFor
\EndFor
\end{algorithmic}
\end{algorithm}
